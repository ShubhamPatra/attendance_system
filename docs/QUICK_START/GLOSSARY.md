# AutoAttendance: Comprehensive Glossary

This glossary defines all technical and project-specific terms used in AutoAttendance documentation. **Organized alphabetically for easy reference.**

---

## A

### Activation Function
A mathematical function applied to neural network neurons. Common types: ReLU (Rectified Linear Unit), Sigmoid, Tanh. Introduces non-linearity enabling networks to learn complex patterns.

**Example in AutoAttendance**: ArcFace uses ReLU activations in ResNet backbone.

### ArcFace
A state-of-the-art deep learning method for face recognition that generates **512-dimensional embeddings**. Uses **additive angular margin loss** to improve class separability.

**Key Property**: L2-normalized embeddings enable fast cosine similarity matching.

**Used in AutoAttendance**: Primary face embedding generator.

### Attention Mechanism
Neural network technique that learns to focus on relevant parts of input data. Common in transformers.

**Example**: Facial attention maps identify important regions (eyes, nose, mouth).

---

## B

### Backbone Network
The main feature extraction component of a deep learning model. In ArcFace, ResNet-100 is the backbone.

**Purpose**: Extract meaningful features from raw images before classification.

### Blink Detection
Eye Aspect Ratio (EAR) based behavioral analysis. Detects eye opening/closing patterns characteristic of live humans.

**Formula**: EAR = (||p2−p6|| + ||p3−p5||) / (2×||p1−p4||)

Where p1-p6 are eye landmark coordinates.

**Used in AutoAttendance**: Part of multi-layer anti-spoofing.

### Bounding Box (BBox)
Rectangular coordinates (x, y, width, height) defining face location in image.

**Output by**: YuNet face detector.

---

## C

### Calibration
Adjustment of model thresholds based on validation dataset performance.

**Example**: Setting RECOGNITION_THRESHOLD=0.38 after testing on 1,000 faces.

### Circuit Breaker Pattern
Design pattern that prevents cascading failures. When database is unavailable, system gracefully degrades rather than repeatedly failing.

**States**: Closed (normal) → Open (failing) → Half-Open (testing recovery)

**Used in AutoAttendance**: MongoDB connection management.

### Confidence Score
Numerical value (0-1) indicating model certainty. Higher = higher confidence.

**Examples**:
- Face detection confidence: Is this really a face? (0.95 = 95% sure)
- Recognition confidence: Is this the enrolled student? (0.92 = 92% sure)

### Convolutional Neural Network (CNN)
Deep learning architecture using convolutional filters. Excellent for image processing.

**Used in AutoAttendance**: YuNet (detection), Silent-Face (liveness), ArcFace (embedding).

### Cosine Similarity
Similarity metric between two vectors based on angle between them. Range: -1 to 1 (1 = identical direction).

**Formula**: cos(θ) = (A · B) / (||A|| × ||B||)

**Why it's better than Euclidean distance for embeddings**: Invariant to scale; works perfectly with L2-normalized vectors.

---

## D

### Database (MongoDB)
NoSQL database storing student data, attendance records, security logs. Flexible schema supports varying embedding dimensions.

**Collections**: students, attendance, courses, users, security_logs, attendance_sessions.

### Deep Learning
Machine learning approach using multi-layer neural networks to learn complex representations.

**Foundation for AutoAttendance**: All core models (YuNet, ArcFace, Silent-Face) are deep learning models.

### Deepfake
Synthesized media (video, audio) created using deep learning. Represents advanced spoofing attack.

**Challenge**: Current anti-spoofing defenses may be vulnerable to high-quality deepfakes.

### Detector
Component that identifies and locates objects in images. In AutoAttendance: **YuNet detects faces**.

**Output**: Bounding boxes + 5-point landmarks.

### Distance Gap
Metric measuring separation between best match and second-best match in recognition.

**Formula**: gap = similarity(1st_best) − similarity(2nd_best)

**Threshold**: ≥0.08 required for high-confidence match (prevents ambiguous cases).

---

## E

### Embedding
Vector representation of data in high-dimensional space. Face embeddings encode facial features as 512 numbers.

**Properties**:
- Similar faces → embeddings close together
- Different faces → embeddings far apart
- L2-normalized to unit length

**Generated by**: ArcFace model.

### Enrollment
Process of capturing student's face samples and generating embedding for database.

**Requirements**: 5-10 face samples from different angles.

### EAR (Eye Aspect Ratio)
See Blink Detection.

---

## F

### FAISS (Facebook AI Similarity Search)
Efficient similarity search library for embeddings. Enables fast face matching even with 10,000+ students.

**Used in AutoAttendance**: Optional vector search for large-scale deployments.

### Face Alignment
Process of normalizing face orientation and size. AutoAttendance standardizes to **112×112 pixels**.

**Purpose**: Consistent input to embedding generator.

### Face Detection
Identifying and locating faces in images. **YuNet** performs this task.

**Output**: Bounding boxes + 5-point landmarks.

### Face Recognition
Identifying *who* a face belongs to by comparing embeddings.

**Process**: Capture → Detect → Align → Generate Embedding → Compare with database → Match or No Match.

### False Negative (FN)
Missed detection—system should have recognized face but didn't.

**Example**: Student marked absent when actually present.

**Impact**: Low attendance accuracy.

### False Positive (FP)
Incorrect detection—system identified non-match as match.

**Example**: Student marked as wrong person.

**Impact**: Serious integrity issue; fraud risk.

### FPS (Frames Per Second)
Video frame processing rate. AutoAttendance targets 30 FPS for smooth real-time operation.

**Hardware dependent**: CPU (10-30 FPS), GPU (30-100+ FPS).

---

## G

### GPU (Graphics Processing Unit)
Specialized processor for parallel computation. Accelerates neural network inference (3-10× faster than CPU).

**AutoAttendance**: Supports optional GPU acceleration (NVIDIA CUDA).

### Graceful Degradation
System continues functioning at reduced capacity when components fail.

**Example**: If liveness model fails, system skips liveness check (higher false positive risk) but continues marking attendance.

### Gradient Descent
Optimization algorithm minimizing loss during model training. Foundation of deep learning training.

---

## H

### Head Movement Detection
Verifying live human by detecting head motion across frames using optical flow analysis.

**Why it works**: Printed photos and screen replays show inconsistent motion patterns.

**Used in AutoAttendance**: Part of multi-layer anti-spoofing.

---

## I

### Inference
Using trained model to make predictions on new data.

**Example**: Feed face image to ArcFace model → Get 512-D embedding.

**Not training**: No weight updates during inference.

### InsightFace
Open-source face recognition library providing pre-trained models (including ArcFace).

**Version used**: 0.7+

**Used in AutoAttendance**: Source of ArcFace weights.

### ONNX (Open Neural Network Exchange)
Model format enabling AI models to run across different frameworks and hardware.

**Advantages**: Portable, optimized inference, GPU/CPU support.

**Used in AutoAttendance**: YuNet distributed as ONNX model.

---

## J

### JWT (JSON Web Token)
Secure token format for authentication. Used for admin/student session management.

**Format**: Header.Payload.Signature

**Used in AutoAttendance**: Role-based access control (RBAC).

---

## K

### KCF Tracker (Kernelized Correlation Filters)
Fast object tracking algorithm. Fallback option if CSRT tracker fails.

**Speed**: Faster than CSRT but less accurate.

**Used in AutoAttendance**: Optional fallback for performance-critical scenarios.

---

## L

### L2 Normalization
Vector normalization to unit length. Results in vectors of length 1.

**Formula**: normalized_vector = vector / ||vector||

**Why it matters**: Enables efficient cosine similarity computation.

**Applied in AutoAttendance**: All face embeddings are L2-normalized.

### Landmarks (Face)
Key facial points (eyes, nose, mouth corners). YuNet detects **5-point landmarks**.

**Purpose**: Face alignment and quality assessment.

### Liveness Detection
Verifying that face is *live human* (not photo/video/mask).

**AutoAttendance method**: Multi-layer (CNN + blink + motion + heuristics).

---

## M

### Machine Learning (ML)
Field of computer science where systems learn patterns from data without explicit programming.

**Application**: Face recognition learns to match faces without hard-coded rules.

### Mask Attack
Spoofing attempt using silicone or 3D-printed mask of enrolled person.

**Challenge**: Physically blocks texture variations.

**Defense**: Multi-layer approach combines CNN + motion detection.

### MongoDB
NoSQL database with flexible schema. Stores AutoAttendance data (students, attendance, logs).

**Advantages for embeddings**: Binary storage (efficient), flexible arrays (variable embedding dims).

### Motion Detection
Identifying moving objects frame-to-frame using optical flow.

**Used in AutoAttendance**: 
- Skips frames with no motion (optimization)
- Detects head movement (anti-spoofing)

### Multi-Frame Voting
Requiring ≥3 of 5 consecutive frames to confirm match before marking attendance.

**Purpose**: Reduces false positives from single noisy frame.

**Confidence threshold**: Only frames with confidence ≥0.42 count.

---

## N

### Neural Network
Interconnected layers of artificial neurons mimicking brain structure.

**Application**: Foundation of all deep learning models in AutoAttendance.

---

## O

### Optical Flow
Technique estimating pixel motion between consecutive frames.

**Used in AutoAttendance**: 
- Motion detection (are there moving faces?)
- Head movement verification (liveness check)

### ONNX Runtime
Library executing ONNX models efficiently on CPU and GPU.

**Used in AutoAttendance**: Runs YuNet face detector.

---

## P

### Precision
Metric measuring fraction of positive predictions that were correct.

**Formula**: Precision = TP / (TP + FP)

**Example**: If system marks 100 faces, and 99 are correct, precision = 99%.

### Pre-training
Training neural network on large general dataset before fine-tuning on specific task.

**Example**: ArcFace trained on VGGFace2 (2.6M faces) before deployment.

### Proxy Attendance Fraud
Student marking attendance on behalf of absent peer.

**Primary problem AutoAttendance solves**: Facial recognition makes impersonation nearly impossible.

---

## Q

### Quality Gating
Filtering out low-quality face samples before recognition.

**Checks**: Blur (Laplacian variance), brightness (HSV V channel), size (relative to frame).

**Purpose**: Prevents false matches from degraded faces.

---

## R

### Recall
Metric measuring fraction of actual positives correctly identified.

**Formula**: Recall = TP / (TP + FN)

**Example**: If 100 students present and system recognizes 99, recall = 99%.

### Recognition Threshold
Similarity score cutoff for confirming identity match.

**Default**: 0.38 (configurable).

**Lower threshold**: More false positives, lower security.

**Higher threshold**: More false negatives, lower usability.

### ResNet-100
Deep residual neural network with 100 layers. Used as backbone in ArcFace.

**Architecture**: Skip connections enabling very deep networks.

**Purpose**: Extract discriminative face features.

### Real-Time Processing
System response time < 100ms, enabling fluid user experience.

**AutoAttendance**: Typical latency 65-95ms per face.

---

## S

### Screen Replay Attack
Spoofing attempt playing video of enrolled student on smartphone/laptop screen.

**Challenge**: Captures dynamic movement.

**Detection**: Multi-layer (motion inconsistency, contrast artifacts, frame reflection patterns).

### Security Log
Audit trail recording every face rejection with reason, confidence, timestamp.

**Purpose**: Investigate false rejections, detect fraud attempts, compliance reporting.

### Silent-Face-Anti-Spoofing
CNN-based liveness detection model. Classifies: Real=1, Spoof=0, Other_Attack=2.

**Training data**: CASIA-CeFA, SiW, OULU-NPU datasets.

**Used in AutoAttendance**: Primary liveness classifier.

### SocketIO
Bidirectional communication protocol enabling real-time updates.

**Use in AutoAttendance**: Live camera feeds, instant admin notifications.

### Softmax Loss
Traditional classification loss function. **Limitation**: Doesn't optimize for margin between classes.

**Alternative**: ArcFace loss uses additive angular margin (superior for embeddings).

### Spoofing
Fraudulent attempt to bypass face recognition using non-live attack (photo, video, mask, etc.).

---

## T

### Temporal Voting
See Multi-Frame Voting.

### Threshold
Numerical cutoff value for decision-making.

**Examples**:
- Recognition threshold: 0.38 (similarity must exceed this)
- Confidence threshold: 0.42 (face detection confidence must exceed this)
- Liveness threshold: 0.50 (confidence in real face must exceed this)

### Tracker (CSRT)
Algorithm maintaining object identity across video frames.

**CSRT**: Discriminative Correlation Filter with Spatial Reliability.

**Used in AutoAttendance**: Tracks students across frames (optimization + continuity).

### Transformer
Neural network architecture using attention mechanisms. Increasingly common in modern vision models.

**Not used in AutoAttendance**: Current pipeline uses CNN-based models for efficiency.

### True Negative (TN)
Correct rejection—system correctly identified non-match.

**Example**: Unknown person correctly rejected.

### True Positive (TP)
Correct match—system correctly identified student.

**Example**: Enrolled student recognized.

### Two-Stage Recognition
Hybrid approach combining fast coarse filter + detailed matching.

**Stage 1** (Fast): Top-K candidates, threshold 0.30.
**Stage 2** (Accurate): Detailed comparison, threshold 0.38.

**Benefit**: 4.7× faster at scale while maintaining 99%+ accuracy.

---

## U

### Unknown Face
Face not enrolled in system. Handled gracefully (queued for manual review or marked as failed).

---

## V

### Vector (Embedding)
Array of numbers representing data in high-dimensional space.

**Face embedding**: 512 numbers representing facial features.

**Properties**: Distance between vectors indicates similarity.

### VGGFace2
Large-scale face recognition dataset (2.6M images, 9,131 subjects).

**Used**: Pre-training ArcFace model.

### Video Replay Attack
Playing recorded video of enrolled student to fool system.

**Detection**: Optical flow inconsistencies, repeated motion patterns, temporal anomalies.

---

## W

### Weighted Loss
Loss function combining multiple objectives with different weights.

**Example**: ArcFace loss = recognition_loss + margin_penalty.

### WebSocket
Protocol for bidirectional communication over single connection.

**Used in AutoAttendance**: SocketIO uses WebSocket for real-time updates.

---

## X

### XAI (Explainable AI)
Techniques making AI model decisions interpretable to humans.

**Application**: Explaining why student was rejected (low confidence, poor liveness score, etc.).

---

## Y

### YuNet
Lightweight single-stage face detector (230KB ONNX model).

**Architecture**: Anchor-free CNN.

**Output**: Bounding boxes + 5-point landmarks.

**Advantages**: CPU-efficient, high accuracy, small model size.

**Used in AutoAttendance**: Primary face detector.

---

## Z

### Zero-Shot Learning
Recognizing unseen classes without training examples.

**Challenge for AutoAttendance**: Requires pre-enrollment (not zero-shot currently).

**Future work**: Enable enrollment-free recognition.
